# 项目需求文档 (Draft)

## 1. 项目概述
本项目旨在开发一个后端服务，提供 RESTful API 接口，用于按需获取 Instagram 用户发布的帖子数据（包括图片、视频、文案等）。系统将部署在 Linux 服务器上。

## 2. 核心功能
*   **按需获取数据**:  不进行全量实时监控，而是根据 API 请求参数（时间范围或条数限制）抓取指定用户的历史/最新帖子。
*   **支持批量用户**:  API 入参支持单个用户名或用户列表。
*   **数据内容**:  (需参考原需求文档，通常包括：帖子链接、发布时间、文案、媒体文件地址/内容、点赞数、评论数等)。

## 3. 技术约束与环境
*   **前端**:  不需要开发前端界面。
*   **交付物**:  RESTful API 接口文档及部署好的后端服务。
*   **部署环境**:  Linux 服务器。
*   **图片/视频访问**:  需要解决内网/本地存储的媒体文件如何对外提供访问的问题 (即 "如何让用户访问图片")。

## 4. 关键问题 (待确认)
1.  **媒体文件访问方案**:
    *   *方案 A*: 爬虫只返回 Instagram 的原始 CDN 链接（优点：不占服务器空间；缺点：链接可能有时效性，且可能需要代理才能访问）。
    *   *方案 B*: 爬虫将图片/视频下载到 Linux 服务器本地，并通过 Nginx 或应用框架静态文件服务对外提供访问（优点：稳定；缺点：占用服务器存储，需要配置域名/IP访问）。
    *   *方案 C*: 爬虫将图片/视频上传到第三方对象存储（如 AWS S3, 阿里云 OSS）（优点：最专业；缺点：有额外成本）。
    *   *当前倾向*: 考虑到“交付给对方集成”，方案 B (本地代理服务) 或 方案 A (如果对方有海外网络环境) 可能最合适。你需要确认**调用方是否在海外环境**以及**对图片时效性的要求**。
2.  **API 响应速度**: 实时爬取通常较慢（秒级甚至分钟级），API 是同步返回结果（用户等很久），还是异步返回（提交任务 -> 轮询状态 -> 获取结果）？
3.  **账号与风控**: 仍然需要考虑 Instagram 的反爬限制。是否提供 Cookie 池或代理 IP 池的支持？

## 五、核心功能（MVP）

### 5.1 监控最新活动

**做什么**：定期追踪各品牌的最新营销动态

**信息来源**：
- 品牌官方 Twitter/X 账号
- 品牌官方 Instagram 账号
- 品牌官网新闻/活动页面

**监控方式**：
1. 定期检查各品牌 SNS 账号的最新发布
2. 筛选出与活动相关的内容（抽奖、展会、快闪等关键词）
3. 自动整理成结构化的活动信息

**输出**：
- 最新活动列表（按时间排序）
- 新活动提醒

---

### 5.3 评估国内适用性

**做什么**：判断活动是否适合在国内落地，给出复用建议

**评估维度**：

| 维度 | 考虑因素 |
|-----|--------|
| **平台可行性** | 国内是否有对应平台（微博 vs Twitter、小红书 vs Instagram） |
| **法规合规性** | 抽奖活动是否符合国内法规（转发抽奖需备案等） |
| **用户习惯** | 国内用户是否习惯这种活动形式 |
| **成本可控** | 活动执行成本是否可接受 |
| **IP/资源限制** | 是否涉及国内无法获取的 IP 或资源 |

**适用性结果**：
- ✅ 适合落地 —— 可直接复用或稍作调整
- ⚠️ 需调整 —— 核心思路可用，但需调整平台/形式
- ❌ 不适合 —— 受限于平台/法规/资源，难以复用

**输出**：
- 适用性评估结果
- 国内落地建议（如果适合）
